暑假提升规划

发表于2026-07-21|更新于2026-07-21|生活规划
|浏览量:

前言

暑假重心是 实习，其他方向不是硬肝，而是利用空闲时间建立起整体认知。

这个暑假不做完人，只做四件事的”了解级”认知：

实习经历 + 算法框架 + 项目思路 + 八股常识

每个方向到什么程度，下面说清楚。

一、实习实践（主线）

暑假的核心。实习经历本身就是最好的竞争力。

需要留意的点

  • 理解企业项目的开发流程（和学校项目不一样在哪）
  • 留意前后端是怎么协作的（API 怎么定、联调怎么搞）
  • Git 团队开发流程（分支、合代码、Code Review）
  • 遇到问题怎么排查（日志、断点、问人）
  • 需求是怎么从业务变成代码的

最终沉淀

实习结束能讲清楚：

  • 这个项目做什么的
  • 我负责了什么
  • 遇到了什么困难、怎么解决的

这就够了。

二、算法 — 建立整体框架

不是刷题冲刺，而是把常见算法类型过一遍，知道怎么回事。

需要了解的算法大类

到什么程度

  • 看到一道题，能判断出 这是什么类型
  • 知道这类题 通常用什么解法
  • 不要求独立 AC，能看着题解理解思路即可

目标：面试聊算法时，不心虚，能说出个一二三。

三、AI 项目 — 有整体思路就行

不是要完整做出来，而是脑子里有一个可以讲的方案。

项目构思要点

  • 找一个有实际场景的方向（结合实习业务或日常需求）
  • 想清楚整体架构：前端 → 后端 → AI 模型怎么串
  • 了解会用到的技术：
    • 大模型调用（API 怎么调、模型怎么选）
    • Prompt 设计（怎么写效果好）
    • RAG 大概流程（文档 → 向量 → 检索 → 生成）
    • Agent 是什么（工具调用、多步推理）

到什么程度

  • 面试时能讲：“我想做一个 XX 项目，大概是这样设计的……”
  • 能用技术名词说出方案（而不是笼统说”用 AI 实现”）
  • 没有实际代码也没关系，思路清晰比做出来更重要

四、八股知识 — 了解级认知

不是背，是建立一个知识地图，知道每个方向大概有什么。

需要了解的板块

到什么程度

  • 别人提到一个名词，你 知道它是什么、用来解决什么问题
  • 不用背细节，但能说出大概原理
  • 能结合实习项目中的实际场景来理解（例如：项目里用了 Redis，它解决的是什么问题）

目标：面试官问起来，能聊几句，不至于一脸懵。

最后

实习已经很累了，其他方向能做到 “知道、了解、能聊” 就已经是很大的进步。

暑假结束的时候，这四样东西每样有个 60 分，就是成功的暑假。
