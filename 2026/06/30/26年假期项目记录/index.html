2026 暑假项目创意记录

发表于2026-06-30|更新于2026-06-30|生活规划
|浏览量:

前言

假期已至，把当下盘旋在脑海里的几个项目创意落下来，避免半途遗忘。整体围绕 “工具自用 + 原理研究 + 赛事实战” 三条线展开，分别对应：自用记事本、AI 查重率机理研究、SkillCreft 与 Trae 参赛项目。

项目一：轻量记事本

定位

一个够用就好的随手记工具，不追求 Notion 那种”全家桶”，而是回归”打开就能写、写完就同步、想看就能置顶”。

核心能力

1. 轻量化

  • 移动端 APK 体积控制在个位数 MB
  • 启动时间 < 1s，本地优先，离线可用
  • 只做纯文本 + 简单 Markdown，不堆富文本功能

2. 手机电脑同步

  • 采用本地 SQLite + 远端增量同步策略
  • 冲突解决以”时间戳 + 最近编辑胜利”为主，复杂冲突提示人工合并
  • 同步协议优先走自建接口，避免第三方限速

3. 界面置顶

  • 笔记级置顶：常用笔记常驻列表顶部
  • 窗口级置顶（桌面端）：参考 Snap/Sketch 风格，单条笔记可拆出小窗口悬浮置顶，方便边查边记

拓展方向：笔记自动上床博客功能

“上床” = 上传到博客（Hexo）的简称，非错别字 😃

  • 在笔记中通过 front-matter 标记 publish: true 触发流程
  • 同步时按 tag/categories 自动归类到 source/_posts/ 对应分类
  • CI 端：推送即触发 hexo clean && hexo g && hexo d，实现”记笔记即发博客”
  • 难点：草稿/正式分离、图片资源迁移、避免误发布临时碎碎念

技术栈预选

  • 桌面端：Tauri（Rust + 前端）/ Electron 二选一，倾向 Tauri
  • 移动端：Flutter 或 React Native
  • 后端：SpringBoot + MySQL/SQLite，对接现有 Hexo 仓库 Git

项目二：理解现存的 AI 查重率计算方式

研究目标

搞清楚目前主流”AI 生成率/查重率”工具（如 GPTZero、Turnitin AI、知网 AIGC 检测等）背后的判定逻辑，写成可复述的笔记 + 复现 demo。

拆解方向

1. 困惑度（Perplexity, PPL）

  • 模型对文本的”意外程度”，AI 生成文本通常 PPL 偏低

2. 突发度（Burstiness）

  • 句长、结构的变化幅度，人类写作更”忽长忽短”

3. Token 概率分布

  • 对每个 token 取 log-prob，统计落在高置信区间的比例

4. 分类器方法

  • RoBERTa / OpenAI 自家 classifier 的微调思路与样本构造

5. 各家差异

  • 知网 AIGC：偏学术语义指纹
  • GPTZero：偏统计特征
  • Turnitin AI：混合分类器，对外黑盒

产出

  • 一篇系统笔记 + 一个最小可跑的 PPL/Burstiness 计算脚本（Python + transformers）
  • 重点不是”击败查重”，而是理解机制，避免无意识的低质 AI 文

项目三：SkillCreft 与 Trae 参赛项目

背景

  • SkillCreft：技能 / Skill 构建类赛事（结合本次 opencode skill 实践）
  • Trae：字节 AI IDE 赛道，重于”在 AI 辅助下完成完整作品”

选题思路

优先选能跑通、能演示、有故事的题，而非追新技术堆叠：

维度 倾向
主题 偏工具型 / 效率型，避免大而全
演示 必须有可交互页面或 CLI
AI 协作痕迹 Trae 侧保留对话 / 代码生成轨迹作为加分项
复用 尽量与项目一（记事本）打通，一码多赛

候选题

  1. AI 驱动的本地知识碎片整理器：基于项目一记事本，加 RAG 检索自笔记
  2. Prompt 优化器：输入 prompt → 反向改写 → 多版本对比
  3. 赛事技能（Skill）工厂：围绕 SkillCreft 主题，做一个可视化生成 opencode skill 的工具

时间线

  • 第 1 周：定题 + 技术选型 + 原型
  • 第 2-3 周：核心功能闭环
  • 第 4 周：演示物料（截图、录屏、readme）

三条线的优先级与交叉

  • 主线：项目三（有死线）
  • 副线：项目一（长期自用）
  • 支线：项目二（研究型，可延后但不放弃）

本假期需解决

  • 记事本桌面端 MVP（新建/编辑/置顶/本地存储）
  • 记事本移动端最小可用版
  • 同步接口设计文档 + 第一版实现
  • AI 查重 PPL/Burstiness 脚本跑通
  • SkillCreft / Trae 定题并提交报名
  • 笔记 → 博客 自动上床链路联调
